# AI Rules Module

Collection of rules, guidelines, and best practices for AI agent development and coordination.

## Overview

This repository contains comprehensive rules and standards for:

-   AI agent behavior and coordination
-   Code generation and documentation standards
-   File management and project structure
-   Writing style and technical documentation
-   Security and compliance practices

## Structure

-   `AGENTS.md` - Agent guidelines and development workflow
-   `CLAUDE.md` - Claude-specific instructions
-   `CONTRIBUTING.md` - Contribution guidelines
-   `.cursor/rules/` - Cursor rules and standards (if applicable)
-   `.github/copilot-instructions.md` - GitHub Copilot instructions
-   `.github/ISSUE_TEMPLATE/` - Issue templates for bug reports and feature requests
-   `.github/PULL_REQUEST_TEMPLATE.md` - Pull request template
-   `SECURITY.md` - Security policy and vulnerability reporting

## Usage

These rules are designed to be used with AI coding assistants (Cursor, Claude, GitHub Copilot, etc.) to ensure consistent, high-quality code generation and documentation.

### GitHub Copilot

To use this repository with GitHub Copilot:

1. **Automatic Detection**: GitHub Copilot automatically reads `.github/copilot-instructions.md` when present in the repository
2. **Reference Files**: Copilot will use the rules in `AGENTS.md`, `CLAUDE.md`, and `.cursor/rules/` to guide its suggestions
3. **Context Awareness**: The rules provide context about coding standards, file management, and project structure
4. **Best Practices**: Follow the guidelines in `CONTRIBUTING.md` when making changes

For more information, see [GitHub Copilot documentation](https://docs.github.com/en/copilot).
